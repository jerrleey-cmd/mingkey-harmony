# 铭键 MingKey 鸿蒙输入法 Agent PoC

## 项目简介

**铭键（MingKey）** 是铭链科技打造的下一代智能输入法，基于链上存证协议的价值互联网入口级产品。核心理念：**键入即服务，一字一链**。

本项目是铭键输入法的鸿蒙HarmonyOS实现PoC（概念验证），展示了：
- 基于IME Kit的输入法Extension架构
- 双进程架构（无网络权限的键盘 + 有网络权限的主应用）
- IPC通信机制（CommonEvent）
- 长按空格键触发翻译Agent

## 核心架构

### 双进程架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        主应用进程 (entry)                        │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────────────┐  │
│  │  EntryAbility │    │ TranslateService│    │  CommonEvent发布  │  │
│  └─────────────┘    └─────────────┘    └─────────────────────┘  │
│         │                  │                     │              │
│         │   有网络权限      │                     │              │
│         │   调用翻译API     │                     │              │
│         └──────────────────┼─────────────────────┘              │
└───────────────────────────┼────────────────────────────────────┘
                            │ IPC (CommonEvent)
┌───────────────────────────┼────────────────────────────────────┐
│                           ▼                                     │
│                 ┌─────────────────────┐                          │
│                 │   翻译请求/响应事件  │                          │
│                 └─────────────────────┘                          │
│                           │ IPC (CommonEvent)                     │
│                           ▼                                      │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │                    输入法进程 (keyboard)                    │ │
│  │  ┌─────────────────────────────┐  ┌────────────────────┐  │ │
│  │  │  InputMethodExtensionAbility │  │  KeyboardComponent │  │ │
│  │  └─────────────────────────────┘  └────────────────────┘  │ │
│  │         │                                  │               │ │
│  │         │    无网络权限                      │               │ │
│  │         │    管理键盘生命周期                 │               │ │
│  │         └──────────────────────────────────┘               │ │
│  │                    键盘UI + 用户输入                         │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                         用户视角                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 隐私设计

1. **本地优先**：键盘模块无网络权限，原始输入内容不离开设备
2. **最小上链**：只有需要AI处理的请求才通过IPC发送
3. **加密通道**：IPC通信数据加密传输
4. **用户可控**：用户可控制哪些场景触发AI服务

## 快速开始

### 环境要求

- Node.js 18+
- HUAWEI DevEco Studio 4.0+
- OpenHarmony SDK API 14

### 构建步骤

1. 克隆项目
2. 使用DevEco Studio打开项目
3. 配置签名信息
4. 点击构建

### 安装测试

1. 连接设备或启动模拟器
2. 在系统设置中启用"铭键输入法"
3. 将铭键设为默认输入法
4. 在任意输入框中测试

## 使用说明

### 基本输入

1. 点击字母键输入英文
2. 点击shift切换大小写
3. 点击123切换到数字键盘
4. 点击#/+=切换到符号键盘
5. 点击空格插入空格

### 翻译功能（PoC演示）

1. 输入中文（如"你好"）
2. 长按空格键500ms
3. 等待翻译结果显示
4. 点击翻译结果插入到文本

## 注意事项

1. **PoC限制**：本版本翻译功能使用模拟数据，仅用于演示
2. **权限说明**：keyboard模块故意不声明网络权限，确保隐私
3. **IPC通信**：entry和keyboard必须同时安装才能正常工作
4. **系统兼容**：需要HarmonyOS 5.0.3或更高版本
